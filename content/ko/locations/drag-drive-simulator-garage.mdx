차량을 최대한 활용하려면 다양한 개조 카테고리를 이해해야 합니다. 커스터마이징 시스템은 여러 섹션으로 나뉘어 있어, 드래그 트랙 위에서 차량의 시각적 매력과 물리적 존재감을 모두 미세 조정할 수 있습니다.

| **보조 재질 (Secondary Material)** | 매트, 메탈릭, 크롬, 글로스 | 차량 페인트 스타일 및 반사 효과 강화 |

<Section icon={Map} title="단계별 차고 커스터마이징 및 저장 가이드">

커스텀 빌드를 만들려면 차고 인터페이스를 탐색하고 저장 시스템을 활용해야 합니다. 차량이 사라지거나 서버를 나갈 경우 저장되지 않은 수정 사항은 손실됩니다. 커스텀 빌드를 영구적으로 저장하려면 다음 단계를 따르세요.

<Steps>
  <Step num="1" title="차고 인터페이스 접속">
    차량을 지정된 차고 구역으로 운전해 들어가거나 플레이어 HUD에서 커스터마이징 메뉴를 열어 편집을 시작합니다.
  </Step>
  <Step num="2" title="페인트 및 재질 선택">
    재질 탭으로 이동하여 기본 및 보조 페인트 마감을 적용합니다. 매트, 메탈릭, 글로스, 크롬 중에서 선택하세요.
  </Step>
  <Step num="3" title="커스텀 휠 장착">
    휠 메뉴를 탐색하여 차량 클래스에 어울리는 디자인을 선택합니다. 스탠스가 드래그 레이싱 선호도에 맞는지 확인하세요.
  </Step>
  <Step num="4" title="커스텀 프리셋 저장">
    차고 인터페이스 내의 저장 메뉴를 열고 빈 슬롯을 선택한 뒤 프리셋을 저장하여 수정 사항을 보호합니다.
  </Step>
</Steps>

저장된 프리셋을 관리하면 드래그 레이싱, 롤플레이, 경찰 추격전 등 상황에 따라 서로 다른 설정을 빠르게 교체할 수 있습니다.

| 프리셋 기능 | 메뉴 명령 | 실제 적용 |
| :--- | :--- | :--- |
| **프리셋 저장** | Save Build | 현재 페인트, 휠, 재질 구성을 저장 |
| **프리셋 불러오기** | Load Build | 저장된 커스텀 차량을 즉시 소환 |
| **차량 초기화** | Clear Customization | 차량을 공장 출고 상태로 되돌림 |

<Callout type="warning" title="저장 경고">
초기화 버튼을 누르거나 다른 차량 클래스를 소환하기 전에 항상 프리셋을 저장했는지 다시 한번 확인하세요. 저장되지 않은 수정 사항은 복구할 수 없습니다.
</Callout>

<Section icon={Target} title="프롭 스포너 및 드래그 모드 설정">

차고에서 만든 빌드를 테스트하기 위해 새로 추가된 저장, 불러오기 및 드래그 모드 프롭 스포너를 사용할 수 있습니다. 이 시스템을 통해 플레이어는 맵 어디에나 커스텀 드래그 트랙과 대기 구역을 건설하여 조직적인 커뮤니티 레이스를 진행할 수 있습니다.

적절한 드래그 스트립을 설정하려면 대기 프롭을 배치하고 개인 서버 명령어를 활용하여 공정한 출발을 보장해야 합니다.

| 도구 / 명령어 | 동작 | 사용 시나리오 |
| :--- | :--- | :--- |
| **프롭 스포너 메뉴** | 트랙 장벽 소환 | 커스텀 드래그 스트립의 경계 설정 |
| **드래그 모드 프롭** | 출발 신호등 배치 | 출발선에 차량 정렬 |
| **!cd [x]** | 카운트다운 시작 | 모든 드라이버의 레이스 시작 동기화 |

드래그 레이스를 개최할 때 채팅창에 `!cd 5`를 입력하여 5초간의 시각적 카운트다운을 화면에 표시하세요. 이는 출발선에서의 혼란을 제거하고 레이스가 네트워크 지연 시간이 아닌 차량 튜닝과 반응 속도에 의해 결정되도록 합니다.

<Callout type="info" title="명령어 정보">
`!cd` 명령어는 변수를 원하는 초 단위 숫자로 변경하여 어떤 기간으로든 조정할 수 있으므로, 다양한 레이싱 형식에 맞춰 고도로 커스터마이징이 가능합니다.
</Callout>
